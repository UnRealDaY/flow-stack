Фаза 2
Realtime сервис — Node.js
Неделя 4
WebSocket сервер + Redis pub/sub WS
socket.io или ws. Redis adapter для горизонтального масштабирования. Аутентификация через JWT при connect.
Спросить об архитектуре ↗
Presence: кто онлайн в workspace
join/leave events, heartbeat каждые 30 сек, TTL в Redis для presence данных.
Events: file:processed, payment:updated Events
Разделение transport (WS) и logic (handlers). Core API публикует в Redis → realtime рассылает клиентам.
Подписка на workspace-specific канал
Клиент подписывается только на свой workspace. Проверка прав при join.
Фаза 3
File processing — Python
Неделя 5
FastAPI + Celery + Redis broker Python
FastAPI для upload endpoint, Celery для async задач, Redis как broker и result backend.
Показать структуру ↗
Pillow: resize, compress, convert Python
Воркер получает задачу → скачивает из S3 → трансформирует → загружает обратно → обновляет статус.
Статусы задач: pending / processing / done / failed State
Храни статус в PostgreSQL. Idempotency key чтобы повторный запрос не создавал дублей.
Retry + dead-letter + уведомление через Redis Reliability
max_retries=3, exponential backoff. После fail → публикуй событие в Redis → realtime уведомит фронт.

Неделя 6
Laravel API + Cashier + планы подписок Laravel
Stateless API (Sanctum token). Plans table: free/pro/enterprise. Cashier для Stripe интеграции.
Про Laravel Cashier ↗
Trial → paid → cancel → refund флоу
Каждый переход — отдельный метод в SubscriptionService. Не смешивай состояния в одном методе.
Webhook обработка с idempotency Critical
Сохраняй stripe_event_id и проверяй перед обработкой. Stripe может слать дубли — это норма.
Laravel Jobs для async billing логики Queues
Webhook → синхронная валидация → Job в очередь → async обработка. Никогда не блокируй webhook ответ.
Событие в Redis после изменения статуса
После успешного payment → публикуй в Redis → realtime сервис уведомляет фронт.
